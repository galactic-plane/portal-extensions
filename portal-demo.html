Portal Inbox Extension

A comprehensive messaging system for Power Pages portals with dual data sources and environment detection.

  • Environment auto-detection (local/portal)
  • Local JSON for development
  • Power Pages Web API for production
  • OData queries ($select, $filter, $orderby)
  • CRUD operations with CSRF auth
  • Read/unread tracking persisted to Dataverse
  • Reply functionality with templates
  • Archive and organize messages
  • External link warnings
  • Fully customizable via manifest

File Structure

Each extension contains exactly 4 files:

  • extension.js - Core logic (deployed)
  • data.json - Local test data (NOT deployed)
  • manifest.json - Config (read by C# plugin)
  • README.md - Documentation

Dual Data Sources

Automatic environment detection:

  • ✓ Local: JSON files (dev/test)
  • ✓ Portal: Web API (production)
  • ✓ OData queries ($select, $filter)
  • ✓ CRUD operations

Generated Init

C# plugin creates initialization:

  • ✓ Reads all manifests
  • ✓ Combines configs
  • ✓ Generates init.js
  • ✓ Deploys to portal

C# Plugin Integration

Automated deployment from manifest

1. C# Plugin reads manifest.json

{
  "initialization": {
    "localDataSource": "messages.json",
    "portalDataSource": {
      "entitySetName": "msfed_messages",
      "baseUrl": "/_api",
      "operations": { /* CRUD config */ }
    },
    "containerId": "portal-inbox-extension",
    "config": { /* Extension config */ }
  }
}

2. Plugin generates portal-extensions-init.js

document.addEventListener('portalExtensionsLoaded', function() {
    PortalInboxExtension.init({
        localDataSource: 'portal-inbox-extension/messages.json',
        portalDataSource: { /* from manifest */ },
        containerId: 'portal-inbox-extension',
        config: { /* from manifest */ }
    });
});

3. Deploy to Power Pages tracker code

<script src="/portal-extensions/portal-extensions.js"></script>
<script src="/portal-extensions/portal-extensions-init.js"></script>

4. Extension auto-detects environment

// localhost → Uses messages.json
// portal domain → Uses /_api/msfed_messages
// Automatic! No manual switching required.
